## 1. 项目概述
### 1.1 项目背景
原有同类工具（OpenLess）采用**云端语音识别接口**，存在网络依赖、隐私泄露、接口配额限制等问题。
本项目基于 **Tauri + Rust + SenseVoice 离线模型** 重构语音识别核心，实现**全离线运行**，保留原有界面、热键、自动粘贴等业务逻辑，适配三大主流桌面系统。

### 1.2 核心功能
- 跨平台麦克风实时音频采集
- SenseVoice-Small ONNX 模型本地推理，中文语音转文字
- 录音启停状态管理、异常捕获与提示
- 与现有 Tauri 前端无缝对接
- 模型内置打包，用户开箱即用，无需额外下载

### 1.3 设计原则
1. **纯离线**：全程本地运算，不发起任何网络请求
2. **高性能**：Rust 底层 + ONNX 推理优化，CPU 低占用
3. **跨平台**：一套代码编译三平台安装包
4. **易维护**：前后端解耦，接口标准化

## 2. 整体技术栈
| 模块 | 技术组件 | 用途说明 |
|------|----------|----------|
| 桌面框架 | Tauri 1.x | 跨平台桌面应用、前后端通信、资源打包 |
| 后端语言 | Rust 1.65+ | 音频采集、模型推理、底层逻辑实现 |
| 音频采集 | cpal 0.15 | 跨平台麦克风音频流采集 |
| 推理引擎 | ort (ONNX Runtime) | 加载并运行 SenseVoice ONNX 模型 |
| 数据处理 | ndarray | 音频数据转为模型所需张量格式 |
| 音频扩展 | rubato | 音频重采样（预留扩展） |
| 前端框架 | Vue / React | 界面渲染、按钮控制、结果展示、全局热键 |
| 离线模型 | SenseVoice-Small (ONNX) | 轻量中文语音识别模型，约80MB |

### 音频硬性规范（模型要求）
- 采样率：`16000 Hz`
- 声道：**单声道**
- 数据格式：`f32 浮点型 PCM`

## 3. 环境搭建指南
### 3.1 通用基础依赖
1. 安装 [Rust 工具链](https://www.rust-lang.org/tools/install)
2. 安装 Node.js 16.x 及以上 + npm
3. 安装 Git

### 3.2 分平台系统依赖
#### Windows
1. 安装 **Visual Studio Build Tools**
2. 勾选组件：**使用 C++ 的桌面开发**
3. 无需额外音频库，cpal 原生兼容 Windows 音频栈

#### macOS
1. 安装 Homebrew
2. 终端执行：
```bash
brew install portaudio
